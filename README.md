# Kapital

![CI](https://github.com/manimovassagh/portfolio/actions/workflows/ci.yml/badge.svg)

> Self-hosted portfolio analytics for Trade Republic — live prices, allocation, performance, tax, and FIRE projections in one dark-mode dashboard.

The repo ships with `exports/sample-portfolio.csv` so you can run immediately without any private data.

## What it does

- **Live prices** — yfinance, refreshed on demand or every 60 s
- **Performance chart** — daily portfolio value, deposits, TWR; 1D / 1W / 1M / YTD / 1Y / Max range selector
- **Allocation** — donut + treemap by type, positions, regions, sectors
- **Analytics** — Sharpe ratio, volatility, max drawdown duration, monthly heatmap, annual P&L bars, geographic exposure
- **Tax** — German FSA (Freistellungsauftrag) tracker with breakdown by dividends, interest, stock perks, realized gains
- **Income** — dividend and interest log with TTM yield per holding
- **Realized P&L** — closed-trade history with per-trade gain/loss
- **Rebalance** — enter target weights, get suggested buy/sell amounts
- **Goals / FIRE** — FIRE number calculator and 10-year projection
- **Watchlist** — persisted watchlist with target prices (SQLite)
- **Markets** — search, quote, history chart, news for any ticker

## Quick start

**Prerequisites:** Go 1.26.1+, Python 3.12+, Node 20+, [`uv`](https://github.com/astral-sh/uv)

```bash
npm install && uv sync   # install dependencies once
make run                 # starts pricer + Go backend (HTTPS) + Vite dev server
```

| URL | |
|---|---|
| `http://localhost:5173` | Vite dev server (frontend) |
| `https://localhost:8766` | Go API backend |
| `http://localhost:8001` | Python pricer (yfinance) |

The browser will warn about the self-signed cert on first run — click **Advanced → Proceed**.

### Make targets

```
make run        Start all three services in the background (logs → logs/)
make stop       Stop all running services
make down       Alias for stop
make logs       Tail live output from all services

make certs      Generate self-signed TLS cert (runs automatically with `make run`)
make install    Install Go, Python, and Node dependencies
make typecheck  TypeScript type-check
make cache      Clear the yfinance price cache
make exports    List CSV files in exports/
```

## Import your data

1. In Trade Republic: **Settings → Export → Transaction history** → download CSV.
2. Drop the file into `exports/`, or use the **Import** button in the app header.
3. If multiple exports exist, pick one from the header dropdown.

## Run with Docker

```bash
make docker        # build images and start stack
make docker-down   # stop and remove containers
make docker-logs   # follow logs
```

After `make docker`:
- Frontend: `http://localhost`
- Go backend: `https://localhost:8766`
- Pricer: `http://localhost:8001`

Place your CSV exports in `exports/` before starting — they are volume-mounted.

### Public demo container

The repository root also contains a single-container Docker build for free hosting on Render.

- Uses the bundled `exports/sample-portfolio.csv` seed only
- Does not require login or persisted watchlist data
- Accepts CSV uploads at runtime, but the container filesystem is ephemeral
- Render terminates TLS, so the deployed service is served over HTTPS

Build locally with:

```bash
docker build --build-arg VITE_PUBLIC_DEMO=true -t portfolio-render-demo -f Dockerfile .
docker run --rm -p 10000:10000 portfolio-render-demo
```

## CI and releases

The GitHub Actions graph runs all checks on pushes and pull requests to `main`:

- Go lint, test, and build
- Python lint and tests
- Frontend lint, tests, build, and Playwright E2E
- Mobile TypeScript check and Expo Doctor
- Final CI Summary gate that fails if any required block fails or is cancelled

Version tags matching `v*.*.*` run the release workflow. The release workflow repeats the lint/test/build checks, builds and pushes the backend, pricer, and frontend Docker images to GHCR, then runs E2E after the images are built.

Publishing a GitHub release also triggers the Docker image workflow so the release packages are rebuilt and tagged from the published release event.

## Prices

- Fetched from Yahoo Finance via `yfinance`, cached for 60 s
- Background refresh every 60 s while the tab is active
- Manual **Refresh** button forces a live fetch
- ISIN → ticker mappings cached in `cache/` (git-ignored)
- Set `PRICE_REFRESH_SECONDS=N` to change the TTL
